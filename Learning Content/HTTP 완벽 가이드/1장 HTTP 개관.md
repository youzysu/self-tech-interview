- 멀티미디어 전송 프로토콜로서의 역할
	- 신뢰성 있는 데이터 전송 프로토콜 (TCP)을 사용
- 월드 와이드 웹(WWW)의 기본 요소: HTTP 클라이언트, HTTP 서버
- MIME(Multipurpose Internet Mail Extensions) 타입: 데이터 포맷 라벨
	1. 사선(/)으로 구분 된 주 타입과 부 타입으로 이루어진 문자열 라벨
- URI (uniform resource identifier) : 통합 자원 식별자
	1. URL: 특정 서버의 한 리소스에 대한 구체적인 위치
		- 표준 포맷
			1. scheme: 리소스에 접근하기 위해 사용되는 프로토콜
			2. 서버의 인터넷 주소
			3. 웹 서버의 리소스
	1. URN: 리소스의 위치에 영향 받지 않는 유일무이한 이름 역할
- HTTP 트랜잭션: 요청 명령과 응답 결과로 구성
- HTTP 메서드 : 서버에게 어떤 동작이 취해져야 하는지 표현
- 상태 코드: 클라이언트에게 요청이 성공했는지 아니면 추가 조치가 필요한지 알려주는 세 자리 숫자
- HTTP 메시지: 정형화된 데이터 덩어리 (단순 줄 단위의 문자열 데이터)
	- 요청 메시지, 응답 메시지
- TCP 커넥션
	- 패킷 교환 네트워크 프로토콜 집합
- 웹의 구성요소: 프락시, 캐시, 게이트웨이, 터널, 에이전트
	1. 프락시: 모든 웹 트래픽 흐름 속에서 신뢰할 만한 중개자 역할
	2. 캐시: 자주 찾는 것의 사본을 저장해두는 프락시 서버
		1. 캐시를 효율적으로 동작하고 최신 버전으로 유지하면서 프라이버시도 보호하기 위한 많은 기능 정의
	3. 게이트웨이: HTTP 트래픽을 다른 프로토콜로 변환
	4. 터널: Raw data를 열어보지 않고 그대로 전달 (HTTP 데이터가 아닌 것을 HTTP 연결에서 그대로 전송해주기 위해 사용)
		- ex. 사내 방화벽 통과 시 활용 (암호화된 SSL 트래픽을 HTTP 커넥션으로 전송)
	5. 에이전트: HTTP 요청을 만들어주는 클라이언트 프로그램
